\section{Vývoj mobilných aplikácií}

\subsection{Prístupy k vývoju mobilných aplikácií}
% Čerpané z: IBM (základná definícia) a Riyas Pullur (technické porovnanie 2025)
Podľa spoločnosti IBM možno proces vývoja mobilných aplikácií definovať ako tvorbu softvéru navrhnutého špecificky pre mobilné koncové zariadenia, pričom kľúčovým rozhodnutím je technológia implementácie. V súčasnosti sa vývojári rozhodujú primárne medzi natívnym a multiplatformovým prístupom, ktoré sa líšia výkonom a náročnosťou na zdroje \cite{pullur_medium_2025}:

\begin{itemize}
    \item \textbf{Natívne aplikácie:} Sú vyvíjané priamo pre konkrétnu platformu (Android, iOS) pomocou dedikovaných nástrojov. Ponúkajú najvyšší výkon a neobmedzený prístup k hardvéru, čo je kľúčové pre aplikácie náročné na systémové zdroje.
    \item \textbf{Webové mobilné aplikácie:} Ide o responzívne webové stránky využívajúce technológie HTML5, CSS a JavaScript, ktoré bežia v prehliadači zariadenia. Modernou formou sú tzv. \textit{Progressive Web Apps} (PWA), ktoré umožňujú inštaláciu na domovskú obrazovku, avšak ich prístup k nízkoúrovňovým funkciám hardvéru (napr. pokročilý geofencing) je limitovaný.
    \item \textbf{Hybridné aplikácie:} Kombinujú predchádzajúce prístupy. Jadro aplikácie je písané vo webových technológiách, ktoré sú následne "zabalené" do natívneho kontajnera. Tento prístup uľahčuje multiplatformový vývoj, no prináša výkonnostnú réžiu spojenú s prekladom volaní medzi webovou a natívnou vrstvou.
\end{itemize}

\subsection{Technické obmedzenia a optimalizácia výkonu}
% Čerpané z: IBM (Cloud Offloading) a Riyas Pullur (Battery & Resource Efficiency)
Vývoj pre mobilné platformy vyžaduje špecifický prístup k správe prostriedkov, ktoré sú v porovnaní s desktopovými systémami limitované. Moderný vývojár sa zameriava na tri kľúčové oblasti:

\begin{enumerate}
    \item \textbf{Energetická a systémová efektivita:} Natívne aplikácie vykazujú vyššiu energetickú efektivitu, čo je kľúčové pri aplikáciách využívajúcich lokalizačné služby (GPS). Efektívna správa prostriedkov priamo ovplyvňuje výdrž batérie zariadenia \cite{pullur_medium_2025}.
    \item \textbf{Cloudové spracovanie (\textit{Cloud Offloading}):} Náročné výpočtové operácie, ako sú komplexné odporúčacie algoritmy, je podľa odporúčaní IBM vhodné delegovať na cloudové servery. Tým sa predchádza preťaženiu lokálneho hardvéru a nadmernému zahrievaniu zariadenia.
    \item \textbf{Odozva používateľského rozhrania (UI):} Natívne frameworky (ako Jetpack Compose) dosahujú najlepšiu plynulosť animácií a rýchlosť odozvy na dotyk. Vďaka priamemu prístupu k vykresľovacím API systému je minimalizované riziko zasekávania rozhrania pri spracovaní dát na pozadí.
\end{enumerate}

\subsection{Toolkit Jetpack Compose}
Jetpack Compose je odporúčaný moderný toolkit spoločnosti Google na budovanie natívneho používateľského rozhrania (UI). 
Zjednodušuje a urýchľuje vývoj UI na systéme Android tým, že umožňuje rýchlo oživiť aplikáciu s menším množstvom kódu, 
výkonnými nástrojmi a intuitívnym API v jazyku Kotlin.

Kľúčové vlastnosti Jetpack Compose zahŕňajú:
\begin{itemize}
    \item \textbf{Menej kódu:} Robte viac s menším množstvom kódu a vyhnite sa celým triedam chýb. Kód je jednoduchý a ľahko udržiavateľný.
    \item \textbf{Intuitívnosť:} Stačí opísať vaše UI a Compose sa postará o zvyšok. Pri zmene stavu aplikácie sa používateľské rozhranie automaticky aktualizuje.
    \item \textbf{Zrýchlenie vývoja:} Je kompatibilný so všetkým vaším existujúcim kódom, takže ho môžete prijať kedykoľvek a kdekoľvek chcete. Umožňuje rýchlu iteráciu pomocou živých náhľadov (\textit{Live Previews}) a plnú podporu v prostredí Android Studio.
    \item \textbf{Výkonnosť:} Compose umožňuje vytvárať atraktívne aplikácie s priamym prístupom k API platformy Android a so vstavanou podporou pre \textit{Material Design}, tmavý režim, animácie a ďalšie funkcie. Rieši deklaratívne UI, prístupnosť (\textit{Accessibility API}) a rozloženie (\textit{Layout}), čím skracuje cestu medzi nápadom a finálnym produktom.
\end{itemize}

\subsection{Programovací jazyk Kotlin}
Kotlin je moderný programovací jazyk, ktorý je stručný, multiplatformový a plne interoperabilný s 
jazykom Java a inými jazykmi. Ide o staticky typovaný jazyk, ktorý využíva viac ako 60 \% profesionálnych
 vývojárov pre Android, pretože pomáha zvyšovať produktivitu, spokojnosť vývojárov a bezpečnosť kódu.

Medzi jeho hlavné výhody patrí:
\begin{itemize}
    \item \textbf{Null Safety (Bezpečnosť voči null hodnotám):} Kotlin výrazne znižuje riziko pádov aplikácií (známy \textit{NullPointerException}) tým, že rozlišuje medzi typmi, ktoré môžu a nemôžu nadobúdať hodnotu \texttt{null} už v čase kompilácie. Aplikácie pre Android používajúce Kotlin majú vďaka tomu o 20 \% nižšiu pravdepodobnosť pádu.
    \item \textbf{Stručnosť a čitateľnosť (Conciseness):} Moderné funkcie jazyka umožňujú písať menej redundantného kódu (\textit{boilerplate}). Príkladom sú dátové triedy (\textit{data class}), ktoré automaticky generujú metódy ako \texttt{equals()} či \texttt{toString()}.
    \item \textbf{Základ pre Jetpack Compose:} Moderný toolkit Jetpack Compose je postavený výhradne na Kotline, čo umožňuje rýchlo vytvárať adaptívne a intuitívne používateľské rozhrania.
    \item \textbf{Interoperabilita:} Kotlin je 100 \% kompatibilný s jazykom Java. To umožňuje v rámci jedného projektu kombinovať súbory v oboch jazykoch a bezproblémovo medzi nimi volať funkcie.
    \item \textbf{Moderné asynchrónne programovanie (Coroutines \& Flow):} Efektívna správa vlákien a práca s prúdmi dát umožňujú vykonávať náročné operácie na pozadí bez blokovania používateľského rozhrania.
    \item \textbf{Smart Casts (Inteligentné pretypovanie):} Kompilátor automaticky sleduje kontroly typov a pretypuje premenné tam, kde je to bezpečné, čím robí kód čistejším a bezpečnejším.
    \item \textbf{Kotlin Multiplatform:} Umožňuje zdieľať biznis logiku kódu naprieč platformami Android, iOS a desktopovými systémami.
\end{itemize}
