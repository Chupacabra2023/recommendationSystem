\subsection{Implementácia mapového modulu a geolokalizácie}                                                                                                                                                                                                                                                                                                                                                                                                         
  Ako základ pre mapu je použitá knižnica \textit{Google Maps SDK}, ktorá poskytuje                                                                                                                                                  všetky potrebné funkcionality pre prácu s geografickými dátami. Používateľ na                                                                                                                                                      hlavnej obrazovke vidí interaktívnu mapu, na ktorej sú jednotlivé udalosti                                                                                                                                                       
  zobrazené vo forme markerov.                                                                                                                                                                                                        
  \subsubsection{Zobrazenie mapy a integrácia s Compose}                                                                                                                                                                           
  Keďže MapView nie je natívny komponent prostredia Jetpack Compose,                                                                                                                                             
  využíva sa AndroidView ako most bridge medzi klasickým
  Android View systémom a Compose UI. Mapa sa pri prvotnom
  načítaní predvolene centruje na súradnice Bratislavy.

  \begin{lstlisting}[language=Kotlin, caption=Inicializácia MapView v prostredí Compose]
  val mapView = remember { MapView(context).also { it.onCreate(null) } }

  AndroidView(factory = { mapView }, modifier = Modifier.fillMaxSize()) { mv ->
      mv.getMapAsync { map ->
          map.moveCamera(CameraUpdateFactory.newLatLngZoom(BRATISLAVA, 13f))
      }
  }
  \end{lstlisting}
  MapView pomocou životných cyklov správne reaguje na udalosti \textit{onPause}, \textit{onResume} a \textit{onDestroy}.


  \subsubsection{GPS poloha a správa oprávnení}
  Po udelení oprávnenia \texttt{ACCESS\_FINE\_LOCATION} aplikácia získa poslednú
  známu polohu zariadenia pomocou \texttt{FusedLocationProviderClient}. Kamera
  sa presunie na polohu používateľa iba raz - tento stav riadi príznak
  \texttt{mapMovedToUser} v \texttt{AppViewModel}, čím sa zabraňuje opakovanému
  presunu kamery pri každej rekomponzícii rozhrania.

  \subsubsection{Práca s markermi a interakcia}
  Markery sú synchronizované so stavom \texttt{appViewModel.activeEvents}
  prostredníctvom \texttt{snapshotFlow} — pri každej zmene zoznamu udalostí
  sa mapa reaktívne aktualizuje. Do vlastnosti \texttt{tag} každého markera
  sa ukladá \texttt{event.id}, čo umožňuje presnú identifikáciu udalosti
  pri kliknutí a následnú navigáciu na obrazovku s detailom.

  \paragraph{Automatické skrývanie skončených udalostí:}
  Vlastnosť \texttt{activeEvents} triedy \texttt{AppViewModel} filtruje udalosti
  tak, aby zoznam obsahoval len tie, ktorých termín konania ešte nenastal:

  \begin{lstlisting}[language=Kotlin, caption=Computed property activeEvents v AppViewModel]
  val activeEvents: List<Event>
      get() = _events.values.filter { event ->
          val end = event.dateTo?.time ?: event.dateFrom?.time
          end == null || end > System.currentTimeMillis()
      }
  \end{lstlisting}

  Skončené udalosti zmiznú z UI
  automaticky bez mazania z databázy - interné \texttt{\_events} zostáva
  kompletné pre potreby histórie a kontroly neúčastí.

  \subsubsection{Okruh pôsobnosti a filtrovanie}
  Markery mimo nastaveného okruhu od používateľa sú automaticky skryté
  (\texttt{marker.isVisible = false}). Okruh je vizualizovaný ako žltý
  polopriesvitný kruh, ktorého polomer je prepojený s nastaveniami
  v \texttt{AppViewModel}.

  \subsubsection{Personalizácia zobrazenia}
  Používateľ môže prepínať medzi piatimi štýlmi mapového podkladu:
  \textit{NORMAL, SATELLITE, HYBRID, TERRAIN} a špeciálnym \textit{DARK}
  režimom využívajúcim vlastnú definíciu štýlu vo formáte JSON.
